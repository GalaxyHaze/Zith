CLI Reference

zith check

Perform syntax and type checking without compilation or linking.

Synopsis

shell
zith check [target] [-v] [--strict]

Description

The check command validates source code for syntax errors and type mismatches without generating machine code. This is useful for rapid feedback during development.

Options

Option Description
-v, --verbose Show detailed type information
--strict Enable strict mode (more warnings)
-q, --quiet Only show errors

Exit Codes

Code Meaning
0 Check succeeded
1 Syntax error
2 Type error
3 File not found

Examples

shell
# Check current project
zith check

# Check specific file
zith check src/main.kali

# Verbose type information
zith check -v

# Strict mode with warnings
zith check --strict

Output Format

Errors are displayed with file, line, column, and message:

text
error: type mismatch
  --> src/main.kali:5:10
   |
 5 | let x: i32 = "string";
   |        ^^^ expected i32, got str

Tips

  • Use check frequently while editing for fast feedback
  • Use --strict to catch potential issues early
  • Integrate with your editor for real-time checking
ℹ
Check is much faster than full compilation and ideal for IDEs and editor plugins.
