CLI Reference

zith build

Compile source files and generate optimized binaries or bytecode with dependency management.

Synopsis

shell
zith build [target] [-m mode] [-o output] [--emit type] [-g]

Description

Builds complete projects by compiling all source files, resolving dependencies, and linking into executable binaries or bytecode. The build command respects configuration in ZithProject.toml.

Options

Option Description Default
-m, --mode MODE Build mode: debug or release debug
-o, --output PATH Output path for binary From toml
--emit TYPE Emit type: bin, lib, or bc bin
-j, --jobs N Parallel jobs Auto
-g Include debug symbols in output -
--check Check only, don't link -
-v, --verbose Verbose output -

Target Resolution

When target is omitted:

Invocation Resolves to
zith build toml.output
zith build -m release Release-optimized build
zith build main.zith -o bin/app bin/app

Build Modes

Debug Mode (default)

  • Minimal optimizations for faster builds
  • Full debug symbols included
  • Assertions and runtime checks enabled

Release Mode

  • Aggressive optimizations
  • Stripped debug symbols
  • Inlining and dead code elimination

Examples

shell
# Build debug binary from project
zith build

# Release build
zith build -m release

# Build specific file
zith build main.zith -o bin/app

# Build with multiple threads
zith build -j 4

# Check only
zith build --check

Output

Successful builds produce:

  • Binary: Executable machine code (default)
  • Library: Shared or static library (with --emit lib)
  • Bytecode: Intermediate representation (with --emit bc)
ℹ
Running zith with no arguments in a directory with ZithProject.toml automatically invokes zith build.
