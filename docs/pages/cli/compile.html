CLI Reference

zith compile

Compile Zith source files directly to machine code or intermediate bytecode.

Synopsis

shell
zith compile <source> [-o output] [--target triple] [--emit type]

Description

Low-level compilation tool that compiles individual source files to machine code or bytecode. Unlike build, this command does not manage dependencies or link multiple files.

Options

Option Description
-o, --output PATH Output file path
--target TRIPLE Target architecture (e.g., x86_64-linux-gnu)
--emit TYPE Output type: asm, obj, bc
-O LEVEL Optimization level (0-3)
-g Include debug symbols

Emit Types

Type Output Use Case
asm Assembly code (.s) Inspection, debugging
obj Object file (.o) Linking with C libraries
bc Bytecode (.bc) Intermediate format

Examples

shell
# Compile to assembly
zith compile main.kali --emit asm -o main.s

# Compile to object file
zith compile app.kali --emit obj -o app.o

# With optimization
zith compile fast.kali -O3 -o fast

# Cross-compile
zith compile app.kali --target aarch64-linux-gnu

Optimization Levels

Level Speed Size Build Time
-O0 Slow Large Fast
-O1 Better Smaller Medium
-O2 Good Good Slower
-O3 Best Larger Slowest
ℹ
For most development, use zith build instead. Use compile only when you need low-level control.
