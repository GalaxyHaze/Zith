Language Guide

Modules & Imports

Organize your code into modules for better structure and reusability.

Module Declaration

Create modules to group related functionality:

kalidous
mod math {
  pub fn add(a: i32, b: i32) -> i32 {
    a + b
  }
}

File-based Modules

Create separate files for modules:

text
src/
├── main.kali
├── math.kali
└── utils.kali

In main.kali:

kalidous
mod math;
mod utils;

Using Modules

Access items with the module path:

kalidous
let result = math::add(5, 3);

Imports

Import items to use them directly:

kalidous
use math::add;
use utils::*;  // Import all

let result = add(5, 3);

Visibility

Control what's accessible outside a module:

kalidous
mod math {
  pub fn public() { }      // Visible outside
  fn private() { }      // Only in module
}

Nested Modules

Create hierarchical module structures:

kalidous
mod graphics {
  pub mod shapes {
    pub fn circle() { }
  }
}

use graphics::shapes::circle;

Re-exports

Export items from submodules:

kalidous
pub use math::add;  // Re-export

Project Structure

Recommended layout:

text
src/
├── main.kali         # Entry point
├── lib.kali          # Library root
├── models.kali       # Data models
├── handlers.kali     # Handlers
└── utils/
    ├── mod.kali      # Module definition
    ├── string.kali
    └── math.kali
ℹ
Keep modules focused on a single responsibility. Aim for modules that are easy to understand and test.
