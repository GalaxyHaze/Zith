Language Guide

Generics

Write code that works with multiple types while maintaining type safety.

Generic Functions

Define functions that work with any type:

kalidous
fn first<T>(arr: [T]) -> T {
  arr[0]
}

let x = first([1, 2, 3]);    // T = i32
let s = first(["a", "b"]); // T = str

Generic Types

Define generic structs and enums:

kalidous
struct Box<T> {
  value: T,
}

let int_box = Box { value: 42 };
let str_box = Box { value: "hello" };

Trait Bounds

Constrain generic types with traits:

kalidous
trait Printable {
  fn display(&self) -> str;
}

fn show<T: Printable>(item: T) {
  println(item.display());
}

Multiple Bounds

Combine multiple trait bounds:

kalidous
fn process<T: Clone + Display>(x: T) {
  // T must implement Clone and Display
}

Associated Types

Types defined in traits:

kalidous
trait Iterator {
  type Item;
  fn next(&mut self) -> Option<Self::Item>;
}

Generic Lifetimes

Generics can include lifetimes:

kalidous
struct Ref<'a, T> {
  value: &'a T,
}

Where Clause

Express complex bounds clearly:

kalidous
fn process<T, U>(x: T, y: U)
where
  T: Clone,
  U: Display,
{
  // Implementation
}

Monomorphization

Generics are specialized at compile time:

kalidous
// This generic function:
fn print_val<T: Display>(x: T) { }

// Gets compiled to:
fn print_val_i32(x: i32) { }
fn print_val_str(x: str) { }
ℹ
Generics have zero runtime cost—they're resolved at compile time through monomorphization.
