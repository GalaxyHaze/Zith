Language Guide

Language Overview

Kalidous is statically typed with a focus on explicit control flow and zero-cost abstractions.

Design Philosophy

Kalidous combines the safety and expressivity of modern languages with the performance and control of systems languages. The design emphasizes:

  • Explicitness: Code should be clear about what it does
  • Safety: Catch errors at compile time, not runtime
  • Performance: Zero-cost abstractions with no hidden overhead
  • Control: Direct access to system resources when needed

Key Features

Static Typing

All types are known at compile time. This catches errors early and enables aggressive optimizations.

Memory Safety

Kalidous provides memory safety without garbage collection through compile-time analysis and explicit ownership semantics.

Modern Syntax

Influenced by contemporary languages, Kalidous syntax is clean and readable while remaining powerful.

Zero-Cost Abstractions

High-level language features compile down to efficient machine code with no runtime overhead.
