Language Guide

Syntax Basics

Kalidous syntax is influenced by modern systems languages, favoring clarity and explicitness.

Hello World

kalidous
fn main() {
  println("Hello, World!");
}

Comments

kalidous
// Single line comment

/* Multi-line
   comment */

/// Documentation comment
/// This appears in generated docs

Statements

Statements are instructions that end with a semicolon. Expressions return values.

kalidous
// Statement
let x = 5;

// Expression (no semicolon)
let y = {
  let x = 3;
  x + 1
};

Blocks

Blocks are enclosed in curly braces and return the value of the last expression:

kalidous
let result = {
  let a = 10;
  let b = 20;
  a + b  // Returns 30
};

Indentation & Formatting

Kalidous uses standard formatting conventions:

  • 2 or 4 spaces for indentation (no tabs)
  • Lines should be under 100 characters
  • Use kalidous fmt to auto-format code

Built-in Functions

Function Purpose Example
println Print line with newline println("hello")
print Print without newline print("hello")
len Length of collection len(array)
ℹ
Statements end with semicolons; expressions do not. This distinction is important in Kalidous.
