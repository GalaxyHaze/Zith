Language Guide

Memory Management

Kalidous provides memory safety through compile-time analysis and explicit ownership semantics.

Ownership

Every value in Kalidous has one owner. When the owner goes out of scope, the value is dropped:

kalidous
fn main() {
  let s = "hello";  // s owns the string
}  // s goes out of scope, memory freed

Move Semantics

Assigning a value transfers ownership:

kalidous
let s1 = "hello";
let s2 = s1;  // Ownership moved to s2
// println(s1);  // Error! s1 no longer owns

Borrowing

Borrow a value without taking ownership using &:

kalidous
fn len(s: &str) -> i32 {
  // Borrows s, doesn't own it
}

let s = "hello";
len(&s);  // Pass reference
// s still owns the string

Mutable Borrows

Borrow mutably with &mut:

kalidous
fn modify(s: &mut str) {
  // Can modify s
}

let mut s = "hello";
modify(&mut s);  // Pass mutable reference

Borrowing Rules

  • One mutable borrow OR multiple immutable borrows (not both)
  • Borrows must not outlive the owner
  • Checked at compile time

Lifetimes

Explicit lifetimes for complex borrowing scenarios:

kalidous
fn longest<'a>(s1: &'a str, s2: &'a str) -> &'a str {
  // Returns reference valid for lifetime 'a
  if len(s1) > len(s2) { s1 } else { s2 }
}

Stack vs Heap

Stack Heap
Fixed size Dynamic size
Fast allocation Slower allocation
Automatic cleanup Managed via ownership
Primitives, arrays Strings, vectors
ℹ
Kalidous's memory model prevents data races and use-after-free errors at compile time.
