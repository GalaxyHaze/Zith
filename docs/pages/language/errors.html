Language Guide

Error Handling

Zith handles errors explicitly through Result types and error propagation.

Result Type

The Result type represents either success or failure:

zith
enum Result<T, E> {
  Ok(T),
  Err(E),
}

Returning Results

Functions can return Result to indicate potential failure:

zith
fn divide(a: i32, b: i32) -> Result<i32, str> {
  if b == 0 {
    Err("Division by zero")
  } else {
    Ok(a / b)
  }
}

Handling Results

Use match to handle results:

zith
match divide(10, 2) {
  Ok(result) => println(result),
  Err(err) => println(err),
}

Error Propagation

Use ? to propagate errors up the call stack:

zith
fn process() -> Result<i32, str> {
  let x = divide(10, 2)?;  // Propagate error
  Ok(x + 5)
}

Option Type

For values that may or may not exist:

zith
enum Option<T> {
  Some(T),
  None,
}

fn find(x: i32) -> Option<i32> {
  if x > 0 {
    Some(x)
  } else {
    None
  }
}

Panic

For unrecoverable errors:

zith
fn unwrap<T>(opt: Option<T>) -> T {
  match opt {
    Some(x) => x,
    None => panic("Called unwrap on None"),
  }
}

Common Patterns

Unwrap with Default

zith
let value = find(-1).unwrap_or(0);  // Returns 0 if None

Map Results

zith
let result = divide(10, 2).map(|x| x * 2);
!
Avoid using unwrap() in production code. Handle errors explicitly instead.
